```raml
#%RAML 1.0
title: ABCAPI
version: 1.0
baseUri: http://api.abccompany.com/{version}
mediaType: application/json

types:
  RedeemRequest: !include types/redeemRequest.raml
  RedeemResponse: !include types/redeemResponse.raml
  ErrorResponse: !include types/errorResponse.raml

/securitySchemes:
  oauth_2_0:
    description: |
      ABCAPI supports OAuth 2.0 for authorizing requests.
    type: OAuth 2.0
    describedBy:
      headers:
        Authorization:
          description: |
            Used to send a valid OAuth 2.0 access token.
          type: string
      responses:
        401:
          description: |
            Bad or expired token. This can happen if the user or ABCAPI revoked or expired an access token. To fix, you should re-authenticate the user.
        403:
          description: |
            Bad OAuth request (wrong consumer key, bad nonce, expired timestamp...). Unfortunately, re-authenticating the user won't help here.
    settings:
      authorizationUri: https://auth.abccompany.com/oauth2/authorize
      accessTokenUri: https://auth.abccompany.com/oauth2/token
      authorizationGrants: [ authorization_code, client_credentials ]

/redeem:
  post:
    description: Pass redeem amount to ABC
    securedBy: [oauth_2_0]
    body:
      application/json:
        type: RedeemRequest
        example: !include examples/redeemRequest.json
    responses:
      200:
        body:
          application/json:
            type: RedeemResponse
            example: !include examples/redeemResponse.json
      400:
        body:
          application/json:
            type: ErrorResponse
            example: !include examples/errorResponse.json
      500:
        description: Internal Server Error
        body:
          application/json:
            type: ErrorResponse
            example: !include examples/errorResponse.json
```

### Breakdown:

#### Metadata:
- **title**: ABCAPI
- **version**: 1.0
- **baseUri**: Base URI where the API is hosted.

#### Types:
- **RedeemRequest**: Defined in a separate file `types/redeemRequest.raml`.
- **RedeemResponse**: Defined in a separate file `types/redeemResponse.raml`.
- **ErrorResponse**: Defined in a separate file `types/errorResponse.raml`.

#### Resources:
- **/redeem**: Endpoint to post redeem requests.

#### Security Schemes:
- **OAuth 2.0**: For securing API access using OAuth 2.0.

#### Responses:
- **200**: Successful response with redeem details.
- **400**: Bad request error response.
- **500**: Internal server error response.

#### Examples:
- Examples for requests and responses are included from external files.

### Modularization:
- Types and examples are placed in separate files (`redeemRequest.raml`, `redeemResponse.raml`, `errorResponse.raml`, `redeemRequest.json`, `redeemResponse.json`, `errorResponse.json`) to keep the main RAML file clean and modular.

### Datatype and Common Error Response:
- Datatype validation and common error responses are defined in separate fragments (`redeemRequest.raml`, `redeemResponse.raml`, `errorResponse.raml`) to ensure reusability and maintainability.
