```raml
#%RAML 1.0
title: ABC API
version: 1.0
baseUri: https://api.abc.com/{version}
mediaType: application/json

types:
  RedeemRequest: !include types/RedeemRequest.raml
  RedeemResponse: !include types/RedeemResponse.raml
  ErrorResponse: !include types/ErrorResponse.raml

/redeem:
  post:
    description: Pass redeem amount to ABC
    body:
      application/json:
        type: RedeemRequest
        example: !include examples/RedeemRequestExample.json
    responses:
      200:
        body:
          application/json:
            type: RedeemResponse
            example: !include examples/RedeemResponseExample.json
      400:
        body:
          application/json:
            type: ErrorResponse
            example: !include examples/ErrorResponseExample.json
    securedBy: [oauth_2_0]

securitySchemes:
  oauth_2_0:
    description: |
      ABC uses OAuth 2.0 for authorization.
    type: OAuth 2.0
    describedBy:
      headers:
        Authorization:
          description: |
            Used to send a valid OAuth 2.0 access token.
          type: string
      responses:
        401:
          description: |
            Bad or expired token. This can happen if the user or ABC revoked or expired an access token. To fix, you should re-authenticate the user.
        403:
          description: |
            Bad OAuth request (wrong consumer key, bad nonce, expired timestamp...). Unfortunately, re-authenticating the user won't help here.
    settings:
      authorizationUri: https://api.abc.com/oauth2/authorize
      accessTokenUri: https://api.abc.com/oauth2/token
      authorizationGrants: [ authorization_code, client_credentials ]
```

This RAML specification defines the API endpoint `/redeem` which accepts a POST request with a JSON body to redeem points. It includes detailed data types and examples for both the request and response, as well as a common error response structure. Security is handled using OAuth 2.0.
